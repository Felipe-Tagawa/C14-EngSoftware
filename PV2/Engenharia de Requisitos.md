
#### Requisitos Funcionais

- O que o sistema DEVE fazer.
- Play, gerenciamento de idioma, gerenciar volume, etc.

#### Requisitos Não-Funcionais

- Sob quais restrições o sistema deve operar(Como fará).
- Desempenho, Espaço, Confiabilidade, Robustez e Usabilidade.

#### Requisitos de Usuários

- User Stories
- Mais alto nível
- Escrito por usuários sem detalhes técnicos.

#### Requisitos de Sistema

- Técnicos
- Escritos pelo developers/Testers
- Requisitos Funcionais e Não Funcionais.

### Elicitação de Requisitos

*Descoberta* (discovery) e entendimento dos requisitos de um sistema. Geralmente feito por um PO;
Deve ter comunicação com o cliente - descobrir quais são os requisitos esperados.
Após a descoberta, deve-se fazer a documentação, verificação, validação e priorização.
A documentação deve ser atualizada de acordo com mudanças de requisitos - evitar perdas ou problemas futuros.

*Rastreabilidade* - após mudanças de requisitos, o código deve ser alterado. Qual User Story foi responsável por mudanças deve estar claro, para evitar demora de achar erros.

### MVP (Produto Mínimo Viável)

- Sistema Funcional Mínimo, que possui um conjunto pequeno de funcionalidades necessárias para comprovar a viabilidade do produto.
- Essencial para perceber falhas a partir de métricas - sucesso, quantidade de usuários e resolução de problemas - teste de viabilidade
- ##### Métricas de Vaidade
	- Métricas que são irrelevantes - ex.: número de visitas em um site

- ##### Métricas Acionáveis
	- Métricas realmente relevantes - ex.: percentual de visitantes que fecharam compras , número de itens comprados
- ##### Métricas de Funil
	- Aquisição (conhecimento da aplicação),
	- Ativação (primeira experiência),
	- Retenção (continua fazendo negócios),
	- Receita (como gerar receita)
	- e Recomendação (clientes recomendam?).
- *Não deve* ser utilizado em casos certos e estáveis.
- Não é exatamente um protótipo, estes não necessariamente são construídos para testar viabilidade de mercado.
- A qualidade deve ser mínima, mas a funcionalidade deve estar funcional.
