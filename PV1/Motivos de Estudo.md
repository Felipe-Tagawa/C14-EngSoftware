#### Testes de Software

Verificam se um programa apresenta um resultado esperado ao ser executado com casos de teste e são divididos em:

Testes Manuais
*Testes Automatizados* - São scripts ou ferramentas que executam verificações em um software sem intervenção humana. Simulam ações do usuário e verificam funcionalidades específicas para garantir que o sistema esteja funcionando de acordo com o esperado.

Tipos
*Unidade* - Validam pequenas partes do código (métodos e funções) - teste do próprio código e não de terceiros;
integração;
performance;
usabilidade.

#### Conceitos importantes (não cai na prova)

1. Defeito

	Código errado, resposta incorreta

2. Bug

	 Termo mais informal, sinônimo de defeito

3. Falha

	Um código com defeito é executado

### Gerência de Configuração - Versionamento

- Todo software é desenvolvido usando um sistema de *controle de versões* (como o git)
- Atua como uma "fonte da verdade" sobre o código
- Permite recuperar versões antigas
- Leitura recomendada: apêndice sobre git

### Processos de Desenvolvimento de Software

- Passos para construção de um software;
- Principais modelos: Waterfall (cascata - mais tradicional) ou ágil - mais recente
- Modelo em cascata: aplicado em modelos de engenharias mais tradicionais, como civil e mecânica, mas software não é como essas outras áreas - Levantamento de requisitos -> análise -> projeto -> codificação -> testes -> implementação; uma coisa atrás da outra;
- Metodologia ágil
